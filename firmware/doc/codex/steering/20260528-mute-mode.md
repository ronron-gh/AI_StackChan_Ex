# Mute mode

## Purpose

外出先での開発・テスト時に AI スタックチャンが TTS や効果音を出さないようにする。
TTS API 呼び出し自体もスキップして料金を節約する。

Web ダッシュボードからトグル切替できるようにし、再起動でリセットされる揮発性。

## Scope

- `firmware/src/main.cpp` — グローバルフラグ `g_mute` と setter/getter
- `firmware/src/Robot.cpp` (or `main.cpp` 呼び出し側) — `robot->speech()` の冒頭で mute チェック
- `firmware/src/main.cpp` — `sw_tone` / `alarm_tone` の冒頭で mute チェック
- `firmware/src/WebAPI.cpp` — `GET /api/mute` / `POST /api/mute` / `/api/status` に mute 状態追加
- `firmware/incbin/dashboard.html` — トグルスイッチ
- `firmware/incbin/dashboard.js` — トグルハンドラ

## Main changes

### main.cpp

```cpp
static volatile bool g_mute = false;
bool is_muted() { return g_mute; }
void set_mute(bool m) { g_mute = m; }
```

### sw_tone() / alarm_tone()

```cpp
void sw_tone() {
  if (is_muted()) return;
  // 既存処理
}
```

### Robot::speech() の TTS 呼び出し箇所

robot->speech() か tts->stream() の冒頭で `if (is_muted()) return;`

### WebAPI

```cpp
void handle_mute_get() { server.send(200, "application/json", String("{\"mute\":") + (is_muted() ? "true" : "false") + "}"); }
void handle_mute_set() {
  String v = server.arg("value");
  set_mute(v == "true" || v == "1");
  handle_mute_get();
}
```

`/api/status` の JSON にも `"mute": true|false` を追加。

### Dashboard UI

- 専用カード「Mute Mode」を追加
- iOS 風トグルスイッチ（CSS のみで実装可能）
- クリックで POST /api/mute?value=true|false

## Design notes

- 揮発性（NVS/SPIFFS 永続化なし）— 外出は一時的、戻ったら自動的に音が戻る方が安全
- スピーカー音量を 0 にする方式（M5.Speaker.setVolume(0)）ではなく、各呼び出し側でガード — TTS の API 呼び出しもスキップして料金節約
- mute 中の LED は通常通り（視覚的フィードバックは維持）

## Verification

1. ビルド成功
2. Dashboard のトグル ON → 額タッチ → AI 応答は来るが**スピーカー無音**、TTS API リクエストも走らない（シリアルログで確認）
3. トグル OFF → 通常通り喋る
4. `curl http://<IP>/api/mute` → JSON 返却
5. `curl -X POST 'http://<IP>/api/mute?value=true'` → ON 切替

## Rollback

- 追加コードを削除、Dashboard から該当カードを削除
- `git revert` で復元可能

## Out of scope

- NVS 永続化（再起動後も状態維持）
- スケジュールベース mute（夜間自動など）
