# Head pet mode

## Purpose

頭を撫でられた（=画面上部から下にスワイプされた）ことを検出し、
気持ちよさそうな表情とリアクション（LED 色変化、簡単な発話）を返す。

M5公式版の `HeadPetModifier` (`../Stackchan/firmware/main/stackchan/modifiers/head_pet.h`) を
コンセプト的に移植する。

## Scope

- `firmware/src/main.cpp`
  - フリック検出の縦方向ブランチに head_pet_trigger() を追加
  - 頭撫で状態管理（3秒後に解除）

## Main changes

### 検出ロジック

既存の loop() 内フリック検出（左右はMod切替）に縦方向の判定を追加：

```cpp
if (t.wasFlicked()) {
  int16_t dx = t.distanceX();
  int16_t dy = t.distanceY();
  notify_activity();
  if (abs(dx) >= abs(dy)) {
    // 左右フリック → 既存の mod 切替（変更なし）
  } else if (dy > 30) {
    // 下方向フリック → 頭撫で
    head_pet_trigger();
  }
}
```

### head_pet_trigger() の振る舞い

- 表情を `Happy` に
- 吹き出しに「えへへ」表示
- LED を ピンク (0xFF60A0)
- サーボでお礼動作（少し上を向く）
- 3秒後に通常状態に戻す（main.cpp の loop() でタイマー監視）

## Design notes

- Avatar クラスに getExpression() が無いため、復帰時は Neutral 固定
- 撫で中に再度撫でられたら、タイマーをリセット
- 会話モード中の撫では避けたい → AiStackChanMod 経由でなく、main で直接処理
- ジェスチャは「下方向 30px 以上のスライド」。閾値はあとで調整可能

## Verification

- ビルド成功
- 画面上部から下にスライド → Happy 表情 + LED ピンク + 吹き出し「えへへ」
- サーボがちょっと上向く
- 3秒後にすべて元に戻る
- 左右フリック（mod 切替）は引き続き動く

## Rollback

## Out of scope
